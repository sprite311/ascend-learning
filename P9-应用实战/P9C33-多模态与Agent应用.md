# P9C33 — 场景三：多模态与 AI Agent 应用

## 引子

大模型的应用正在从纯文本向**多模态**（文本+图像+语音）和 **AI Agent**（自主决策和工具调用）方向发展。在昇腾上部署这类应用有哪些特殊考虑？

---

## 1. 多模态模型部署

### 1.1 典型架构

多模态模型（如 Qwen-VL、InternVL）通常由以下几部分组成：

```
图像 → [视觉编码器 (ViT)] → 视觉 Token
                                   ↘
文本 → [Tokenizer] → 文本 Token  → [大语言模型] → 输出
```

在昇腾上部署的关键点：
- **视觉编码器**：通常是一个 CV 模型（ViT），走标准的张量计算
- **语言模型**：走 LLM 推理引擎（MindIE/vLLM）
- **对齐层**：连接视觉和语言的投影层

### 1.2 特殊挑战

| 挑战 | 说明 |
|------|------|
| 内存占用更大 | 需要同时加载视觉编码器和语言模型 |
| 计算模式混合 | 视觉部分 Compute-Bound，语言部分 Memory-Bound |
| 输入预处理 | 图像需要在 CPU 上裁剪、缩放、归一化 |
| 长上下文 | 图像 Token 可能占数百到数千个 Token |

---

## 2. AI Agent 应用

### 2.1 什么是 AI Agent

AI Agent 是能够**自主使用工具**的 AI 应用。它读取用户指令，决定调用哪些工具（搜索引擎、代码执行器、数据库查询等），然后综合工具返回的结果做出回答。

### 2.2 Agent 对推理引擎的要求

| 要求 | 原因 |
|------|------|
| **低延迟 TTFT** | Agent 需要快速做出决策 |
| **长上下文** | Agent 的上下文包含工具调用历史，通常很长 |
| **高并发** | Agent 可能同时调用多个工具 |
| **流式输出** | 用户需要实时看到 Agent 的思考过程 |

### 2.3 在昇腾上运行 Agent

Agent 框架（如 LangChain、AutoGen）本身不依赖特定硬件。它们通过 OpenAI 兼容的 API 调用底层 LLM。

所以部署 Agent 应用在昇腾上的方式是：
1. 用 MindIE 或 vLLM-Ascend 部署底层 LLM 服务
2. Agent 框架通过 HTTP API 调用 LLM
3. LLM 的部署和优化与普通 LLM 推理完全一致

---

## 本章小结

| 要点 | 内容 |
|------|------|
| 多模态 | 视觉编码器 + 语言模型组合部署 |
| AI Agent | 通过 OpenAI API 接入昇腾上的 LLM 服务 |
| 关键挑战 | 内存更大、长上下文、低延迟要求 |

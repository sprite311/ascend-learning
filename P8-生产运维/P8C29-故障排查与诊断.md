# P8C29 — 故障排查红宝书：从 NaN 到 OOM 的系统化诊断

## 1. 排查原则：分层定位

遇到问题时，按照软件栈的层次**自上而下逐层排查**：

```
1. 先查应用层 → 代码逻辑/输入数据有没有问题？
2. 再查框架层 → 模型在 CPU 上能不能正确运行？
3. 然后查 CANN → 特定算子在 NPU 上是否有精度问题？
4. 最后查驱动/硬件 → 设备是否正常？温度、链路是否异常？
```

---

## 2. 常见故障速查表

### 2.1 精度类

| 现象 | 可能原因 | 排查步骤 |
|------|---------|---------|
| 输出全是 NaN | FP16 数值溢出 | 改用 FP32 测试；检查 loss 是否异常大 |
| 输出偏差大 | 算子精度实现差异 | 逐层对比 CPU vs NPU 输出 |
| 生成乱码 | 权重加载不完整/错误 | 检查权重文件完整性和格式 |

### 2.2 内存类

| 现象 | 可能原因 | 排查步骤 |
|------|---------|---------|
| OOM (Out of Memory) | 模型+KV Cache 超出 HBM | 减小 batch/seq_len；用量化；增加 TP |
| 内存持续增长 | 内存泄漏 | 检查是否有未释放的张量 |
| 碎片化导致分配失败 | 反复分配/释放不规则大小 | 使用内存池；减少动态分配 |

### 2.3 运行时类

| 现象 | 可能原因 | 排查步骤 |
|------|---------|---------|
| 算子不支持 | CANN 版本太旧 | 升级 CANN；用标准算子替代 |
| 卡死/无响应 | 死锁或通信超时 | 检查 HCCL 连通性；检查进程数配置 |
| 设备找不到 | 驱动问题 | `npu-smi info` 检查设备状态 |
| 性能骤降 | 温度过高导致降频 | 检查散热；`npu-smi` 看温度 |

---

## 3. 关键排查工具

| 工具 | 用途 |
|------|------|
| `npu-smi info` | 查看设备状态、温度、显存使用 |
| `npu-smi log` | 查看设备错误日志 |
| `msprof` | 性能 Profiling |
| `msaccucmp` | 精度对比 |
| `dmesg` | Linux 内核日志，查看底层硬件错误 |

---

## 4. 经验法则

1. **先在 CPU 上验证**：如果 CPU 上也有问题，和昇腾无关
2. **最小复现**：用单算子或单层复现问题，排除干扰
3. **检查版本**：70% 的"莫名其妙"问题是版本不匹配导致的
4. **看日志**：CANN 的日志通常会包含有用的错误信息

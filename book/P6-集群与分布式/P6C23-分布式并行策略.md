# P6C23 — 分布式并行策略：TP、PP、EP 的组合艺术

## 引子

当模型太大一张卡放不下，或者想加速训练/推理时，就需要多卡并行。但"并行"不是简单的"把工作分给多张卡"——怎么切分、怎么通信、怎么组合，决定了整体效率。

---

## 1. 四种并行策略详解

### 1.1 数据并行（DP）

每张卡都有完整模型，处理不同数据。

```
卡0: 模型(全) + 数据批次A → 梯度A ─┐
卡1: 模型(全) + 数据批次B → 梯度B ─┼→ AllReduce → 平均梯度 → 更新
卡2: 模型(全) + 数据批次C → 梯度C ─┘
```

- **优点**：实现简单，线性加速
- **缺点**：每张卡都要完整模型，不节省显存
- **通信**：每步一次 AllReduce（梯度同步）

### 1.2 张量并行（TP）

把一个算子的权重矩阵切分到多张卡。

```
权重矩阵 W [4096 × 4096]
├── 卡0: W[:, :2048]     ← 前一半列
└── 卡1: W[:, 2048:]     ← 后一半列

各自计算后 AllGather 合并结果
```

- **优点**：线性减少显存
- **缺点**：每一层都要通信，要求极高带宽
- **通信**：每层两次 AllGather/ReduceScatter
- **建议**：只在机内 HCCS 互联的卡之间使用

### 1.3 流水线并行（PP）

把模型按层切分到不同卡。

```
卡0: Layer 0-9   → 中间结果 →
卡1: Layer 10-19 → 中间结果 →
卡2: Layer 20-31 → 最终输出
```

- **优点**：通信量小（只在切分点传中间结果）
- **缺点**：存在"气泡"（Pipeline Bubble，某些阶段卡在等待上游结果）
- **通信**：只在相邻级之间，数据量等于一个 mini-batch 的激活值

### 1.4 专家并行（EP）

专门用于 MoE（Mixture of Experts）模型（如 DeepSeek-V3）。

MoE 模型中每一层有多个"专家"网络，每个 Token 只激活其中少量专家。不同专家放在不同卡上。

```
卡0: Expert 0, 1      ← Token A 被路由到 Expert 1
卡1: Expert 2, 3      ← Token B 被路由到 Expert 3
卡2: Expert 4, 5      ← Token C 被路由到 Expert 4

通信：AllToAll（把 Token 发送到对应专家所在的卡）
```

- **通信**：AllToAll 原语，通信模式不规则
- **挑战**：负载不均衡（某些专家被激活得更频繁）

## 2. 混合并行策略

实际部署中，通常组合使用多种并行策略：

### 2.1 TP + PP（大模型推理）

```
4 机 × 8 卡 = 32 张卡

机内 8 卡: TP = 8（张量并行）
跨机 4 组: PP = 4（流水线并行）
```

- TP 放机内（通信密集，走 HCCS）
- PP 放跨机（通信少，走 RoCE）

### 2.2 DP + TP + PP（大模型训练）

```
8 机 × 8 卡 = 64 张卡

TP = 8（机内）
PP = 4（4 级流水线，每级 2 机）
DP = 2（2 组数据并行）

8 × 4 × 2 = 64
```

### 2.3 选择策略的依据

| 因素 | 推荐策略 |
|------|---------|
| 模型放不下单卡 | 先 TP，再 PP |
| 想加速训练 | DP |
| MoE 模型 | EP + TP |
| 机内卡数多 | TP 优先（利用 HCCS） |
| 机器数多但机内卡少 | PP + DP |

---

## 本章小结

| 要点 | 内容 |
|------|------|
| 四种并行 | DP（数据）、TP（张量）、PP（流水线）、EP（专家） |
| TP 特点 | 每层通信，要求高带宽，用机内 HCCS |
| PP 特点 | 只在切分点通信，可跨机，但有气泡 |
| 混合并行 | 实际部署通常组合使用，TP 机内 + PP 跨机 |

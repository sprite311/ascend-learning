# P6C24 — 千卡集群部署实战：从机柜到服务

## 引子

部署千卡级的昇腾集群涉及硬件规划、网络设计、软件配置等多个环节。本章梳理关键考虑点和决策逻辑。

---

## 1. 硬件规划

### 1.1 计算节点

每台 Atlas 800 服务器包含 8 颗 Ascend 910B，机内通过 HCCS 全互联。

| 规模 | 服务器数 | NPU 总数 | 适合的模型规模 |
|------|---------|---------|-------------|
| 小型 | 1-4 台 | 8-32 | 7B-72B 推理或微调 |
| 中型 | 8-16 台 | 64-128 | 72B-200B 训练 |
| 大型 | 32-128 台 | 256-1024 | 200B+ 预训练 |

### 1.2 网络设备

机间互联通常采用 **二层叶脊（Leaf-Spine）拓扑**：

```
        [Spine 交换机]  [Spine 交换机]
       /  |  |  \     / |  |  \
      /   |  |   \   /  |  |   \
  [Leaf1][Leaf2][Leaf3][Leaf4]
   |  |   |  |   |  |   |  |
  S0  S1  S2  S3  S4  S5  S6  S7  (服务器)
```

关键指标：
- **收敛比**：建议 1:1（无收敛），确保任意两台服务器间的带宽一致
- **端口带宽**：100GbE 或 200GbE RoCE
- **交换机延迟**：越低越好

### 1.3 存储

模型权重文件、训练数据、checkpoint 需要高性能共享存储：
- 推荐使用并行文件系统（如 Lustre、GPFS）
- 模型加载速度直接影响启动时间和故障恢复时间

## 2. 软件配置

### 2.1 操作系统

支持的 Linux 发行版：
- Ubuntu 20.04/22.04
- CentOS 7.6+（部分版本）
- EulerOS（华为自研 Linux）

### 2.2 软件栈安装（每台机器）

按顺序安装：
1. NPU Driver + Firmware
2. CANN Toolkit + NNAE
3. PyTorch + torch_npu（或 MindSpore）
4. 推理引擎（MindIE 或 vLLM）
5. 通信库 HCCL

**强烈建议使用容器**：华为提供了预置所有依赖的 Docker 镜像，一条命令拉取即可。

### 2.3 网络配置

```bash
# 检查 NPU 设备
npu-smi info

# 检查 HCCL 通信连通性（类似 NCCL test）
hccl_test -b 8M -e 128M -f 2 -g 8

# 配置 RoCE 网卡
# 确保 RDMA 功能开启，MTU 设置为 9000（Jumbo Frame）
```

## 3. 故障域与高可用

### 3.1 故障域设计

大集群中故障是常态（卡死、NPU 故障、网络抖动）。设计时要考虑：

- **节点级隔离**：单台服务器故障不影响集群运行
- **Checkpoint 策略**：定期保存训练状态，故障后从 checkpoint 恢复
- **弹性训练**：支持动态增减节点（MindSpore 等框架支持）

### 3.2 监控

- 使用 `npu-smi` 监控每张卡的温度、功耗、利用率
- 部署 Prometheus + Grafana 做集群级监控
- 关键告警：温度过高、HBM 利用率 100%、通信超时

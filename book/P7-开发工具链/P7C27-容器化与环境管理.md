# P7C27 — 容器化与环境管理：昇腾开发的最佳实践

## 1. 为什么容器化对昇腾特别重要

昇腾的软件栈有严格的版本依赖关系：Driver ↔ CANN ↔ torch_npu ↔ PyTorch，任何一层版本不匹配都可能导致运行失败。

容器化（Docker）的价值在于：**把整套经过验证的软件环境打包成一个不可变的镜像**。不管物理机上装了什么，启动容器就能获得确定性的运行环境。

---

## 2. 昇腾 Docker 使用要点

### 2.1 设备映射

NPU 不像 NVIDIA GPU 那样有成熟的 nvidia-docker runtime。昇腾容器需要手动映射设备文件：

```bash
docker run -it \
    --device=/dev/davinci0 \      # 映射 NPU 设备
    --device=/dev/davinci_manager \
    --device=/dev/devmm_svm \
    --device=/dev/hisi_hdc \
    -v /usr/local/Ascend/driver:/usr/local/Ascend/driver:ro \  # 挂载驱动
    ascend-pytorch:latest
```

### 2.2 华为提供的官方镜像

华为在 Ascend Hub 上提供了预构建的 Docker 镜像：

| 镜像类型 | 包含内容 |
|---------|---------|
| CANN 基础镜像 | Driver + CANN Toolkit |
| PyTorch 开发镜像 | CANN + PyTorch + torch_npu |
| MindSpore 开发镜像 | CANN + MindSpore |
| 推理引擎镜像 | CANN + MindIE 或 vLLM |

### 2.3 版本矩阵管理

建议维护一份团队内部的版本配套表：

```yaml
# environments.yaml
production_v1:
  driver: 24.1.rc1
  cann: 8.0.RC2
  pytorch: 2.3.1
  torch_npu: 2.3.1
  vllm: 0.6.0
  image: "ascend-hub.huawei.com/public-ascendhub/pytorch:24.0.RC2-A2"
```

## 3. CI/CD 集成

### 3.1 自动化测试

```yaml
# GitHub Actions / GitLab CI 示例
test_on_npu:
  stage: test
  image: ascend-pytorch:latest
  script:
    - python -c "import torch_npu; print(torch.npu.is_available())"
    - pytest tests/ --device=npu
```

### 3.2 镜像构建流水线

```
基础镜像 (CANN) → 安装框架 → 安装应用依赖 → 验证测试 → 推送镜像仓库
```

前言
这本书为谁而写
如果你是一名 AI 开发者、算法工程师或技术管理者，正在思考以下问题中的任何一个，这本书就是为你准备的：

"昇腾到底是什么？是一颗芯片，还是一套软件，还是一个平台？"
"我的 PyTorch 模型能不能直接在昇腾上跑？需要改多少？"
"CANN、MindSpore、MindIE、torch_npu……这些名词之间是什么关系？"
"昇腾和 NVIDIA 的技术路线有什么本质区别？"
"我想在昇腾上部署大模型，应该从哪里开始？"
简单来说，这是一本帮助你建立昇腾技术体系完整认知的书。

这本书不是什么
这本书不是一本 API 手册或安装教程。你不会在这里找到"第一步输入什么命令、第二步点击什么按钮"这样的操作指南（那类内容放在了附录和官方文档中）。

这本书要做的是：让你读完之后，脑子里有一张清晰的地图。当你遇到任何昇腾相关的技术问题时，你知道它属于地图上的哪个区域，知道它和其他区域之间的关系，知道为什么会出这个问题，知道应该从哪里入手解决。

写作理念
本书遵循三个核心原则：

1. 通俗易懂——用类比建立直觉
每一个技术概念，我们都会先用日常生活中的类比让你建立直觉。不是为了"有趣"，而是因为人的大脑天生善于理解具体事物。先有直觉，再看细节，学习效率会高很多。

2. 因果链驱动——讲清"为什么"
对于每一个设计决策，我们不仅告诉你"是什么"和"怎么做"，更要深挖"为什么这样设计"。

例如：为什么 CANN 软件栈要分那么多层？不是因为华为的工程师喜欢复杂——而是因为芯片只认二进制指令、AI 框架五花八门、用户需求千变万化，分层是应对复杂度的唯一武器。当你理解了"为什么"，那些看起来繁琐的层次结构就会变得顺理成章。

3. 结构化认知——构建知识体系
本书按照"硬件 → 系统软件 → 框架 → 引擎 → 应用"的自底向上顺序组织。每个 Part 都在前一个 Part 的基础上搭建。读完之后，你的知识不是零散的碎片，而是一棵有根有叶的完整大树。

如何阅读本书
全景入门（半天）
如果你只想快速了解昇腾是什么，请阅读：P1 全部（3 章）→ P3C08（软件栈分层）→ P5C17（推理 vs 训练）

系统学习（1-2 周）
按顺序从 P1 读到 P8，跳过不感兴趣的细节章节。

实战驱动（按需）
如果你已经有昇腾使用经验，直接跳到 P9（应用实战）或 P5（推理部署），遇到不理解的概念再回头查阅对应章节。

关于 Transformer 基础
本书假设读者已经对 Transformer 架构有基本了解。如果你需要系统学习 Transformer，推荐先阅读姊妹篇《Transformer 架构：从直觉到实现》。本书在需要时会简要引用 Transformer 的相关概念，但不会重复讲解。

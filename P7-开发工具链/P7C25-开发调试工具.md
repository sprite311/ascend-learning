# P7C25 — 开发调试工具链：MindStudio 与 Profiler

## 1. MindStudio：昇腾的集成开发环境

MindStudio 是华为为昇腾开发者提供的 IDE，基于 IntelliJ 平台。它集成了算子开发、模型转换、性能分析、精度比对等功能。

### 核心功能

| 功能 | 说明 |
|------|------|
| **算子开发** | Ascend C 算子的创建、编译、测试 |
| **模型转换** | 图形化的 ATC 模型转换流程 |
| **性能分析** | 可视化的 Profiling Timeline |
| **精度比对** | NPU 结果与 CPU 基准的逐层对比 |
| **远程开发** | 连接远程昇腾服务器进行开发 |

在早期学习和算子开发阶段有用，但大多数生产场景下的开发者更倾向于使用命令行工具。

## 2. msprof：性能分析利器

msprof 是昇腾的命令行性能分析工具，对标 NVIDIA 的 Nsight Systems。

### 2.1 使用方式

```bash
# 采集性能数据
msprof --application="python inference.py" --output=./prof_data

# 解析生成报告
msprof --export=on --output=./prof_data
```

### 2.2 关键分析维度

| 维度 | 查看什么 | 判断方法 |
|------|---------|---------|
| **算子耗时** | Top-K 算子排名 | 前 5 名是否合理 |
| **Cube 利用率** | 计算单元占比 | 低于 30% → 可能 Memory-Bound |
| **HBM 带宽** | 搬运速率 | 接近峰值 → Memory-Bound 确认 |
| **Stream 间隙** | 计算间的空闲时间 | 间隙大 → Host 瓶颈或同步过多 |
| **TransData 占比** | 格式转换开销 | 超过 10% → 需要优化数据布局 |

### 2.3 性能分析方法论

建立科学的 Profiling 习惯比盲目调参重要得多：

1. **先看全局**：端到端延迟中 Host vs Device 的占比
2. **锁定 Device**：看 Top-5 算子
3. **分析 Top-1**：判断是 Compute-Bound 还是 Memory-Bound
4. **有针对性优化**：Memory-Bound 就去做量化/融合；Compute-Bound 就增大 Batch

## 3. 精度对比工具

当模型在 NPU 上的结果和 GPU/CPU 不一致时，CANN 提供精度比对工具来定位问题算子：

```bash
# 分别在 CPU 和 NPU 上导出每层的输出 tensor
# 工具自动比对并标记差异超阈值的层
msaccucmp.py -m cpu_dump/ -g npu_dump/ -o compare_result/
```
